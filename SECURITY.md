# Security Policy

## Supported Versions

This repository does not publish versioned releases. Security fixes are applied
only to the latest revision on the `master` branch.

| Version | Supported |
| --- | --- |
| Latest revision on `master` | :white_check_mark: |
| Older revisions and forks | :x: |

## Reporting a Vulnerability

Please do not report suspected security vulnerabilities in a public GitHub
issue.

Report vulnerabilities privately using
[GitHub private vulnerability reporting](https://github.com/ryroeu/vmware/security/advisories/new).

Please include:

- The affected script and commit hash
- A description of the vulnerability and its potential impact
- Steps or proof-of-concept code that reproduce the issue
- Relevant PowerShell, PowerCLI, vCenter, and operating-system versions
- Any suggested remediation, if available

We aim to acknowledge reports within seven days and will provide status updates
while the report is investigated. Please allow reasonable time for a fix before
publicly disclosing the vulnerability.

This policy covers the scripts maintained in this repository. Vulnerabilities
in VMware or Broadcom products should be reported directly to the applicable
vendor.
